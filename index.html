Invite a Collaborator on GitHub using Manage Access

Share exactly one repository—keep everything else private. This step-by-step guide follows GitHub’s Settings → Manage access flow.

Get Started Download PDF

Step-by-Step

Expand each step below. All components are keyboard and screen-reader friendly.

  1. Step 1 Go to your repository

    Sign in at github.com and open the specific repository you want to share, e.g. https://github.com/yourusername/your-repo-name.

  2. Step 2 Open Settings → Manage access

    From the repository top bar, select Settings. In the left sidebar, choose Manage access.

  3. Step 3 Invite a collaborator

    Select Invite a collaborator. In the dialog, search by GitHub username or email, then click Add.

  4. Step 4 Set the right permission

    After they accept, assign the minimal role required for their work. You can change it anytime.

  5. Step 5 Verify, modify, or remove access

    Return to Settings → Manage access to adjust roles or remove collaborators with the ⋯ menu.

Permission Levels

Start conservative. You can always increase access later.

Access Level What they can do Best for
Read View and clone the repository Reviewers, readers
Triage Manage issues and pull requests QA, moderators
Write Push commits, create branches Active contributors
Maintain Manage settings (not delete repo) Project leads
Admin Full control, including managing collaborators Trusted co-owners

Privacy Tips

  • Inviting someone to one repository does not expose your other private repositories.
  • Use the least-privilege role that enables their job.
  • Rotate or remove access when the collaboration ends.
  • Consider organization teams if you routinely add multiple collaborators.
